# 账号监控池分析工具 MVP

这是一个本地网页工具，用于把手动收集到的账号、视频链接、截图备注和文案线索，分析后写入账号监控池 CSV。

## 启动

在项目根目录执行：

```powershell
npm start
```

启动后打开：

```text
http://localhost:4173
```

如需换端口：

```powershell
$env:PORT=4300
npm start
```

## 录入流程

1. 填写平台和账号名称。
2. 账号链接或代表爆款链接至少填写一个。
3. 可点击“填充测试数据”快速跑通流程。
4. 勾选“适合学习什么”的多选项，也可以让工具根据内容自动补充学习项。
5. 点击“生成分析预览”，先确认账号类型、双账号适配理由、监控等级和下一步补充信息。
6. 确认无误后点击“确认入库”，输入和分析字段会追加到 CSV。

## 保存位置

默认保存到：

```text
data/monitoring/account_monitor_pool.csv
```

目录不存在时，服务端会自动创建。CSV 使用 UTF-8 BOM，方便 Excel 直接打开中文。

页面底部会显示最近 50 条已入库账号，方便快速核对是否写入成功。

## 半自动监控工作台

页面新增 5 个入口：

- 账号监控池
- 今日监控任务
- 今日观察记录
- 爆款拆解池
- 每日监控日报

使用顺序：

1. 先在账号监控池里维护账号。
2. 点击“生成今日监控任务”，工具会读取 `account_monitor_pool.csv`，按 A重点、B观察、C备用生成今日建议查看账号。
3. 在任务行点击“补充观察”，打开平台后手动补充今天发现的视频链接、标题、数据、文案、截图和变现痕迹。
4. 点击“保存今日观察记录”，工具会分析并写入 `daily_monitor_records.csv`。
5. 如果视频值得拆，会自动加入 `viral_analysis_queue.csv`。
6. 点击“生成/刷新日报”，工具会生成 `outputs/daily_monitor_report_YYYY-MM-DD.md`。

当前仍然不自动抓取平台数据，不绕过登录、验证码或风控。

## 当前分析逻辑

第一版不做复杂爬虫，不自动访问平台链接，不绕过登录、验证码或平台限制。

分析基于本地规则引擎，围绕双账号定位做关键词评分：

- AI先锋：流量、信任、转化、私信、直播承接、799元AI口播智能体成交。
- AI先锋者：高认知表达、激进观点、行业判断、AI变现观察、长期人设信任。

如果链接无法访问或信息不足，工具会提示继续补充截图、视频文案、点赞评论数据、主页简介和代表视频链接。

## 后续插件升级方向

1. 保留当前服务端 CSV 入库接口，浏览器插件只负责把当前页面链接、标题、选中文案和截图路径传回本地服务。
2. 插件增加右键菜单：“加入账号监控池”“加入代表爆款样本”。
3. 插件只读取用户主动触发的当前页面信息，不做后台批量抓取。
4. 遇到登录、验证码、权限页时，继续走人工补充，不绕过平台限制。
5. 后续可把本地规则引擎替换或增强为可配置 LLM 分析，但 CSV 字段保持兼容。
